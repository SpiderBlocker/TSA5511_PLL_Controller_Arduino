TSA5511 PLL Controller for Arduino

A practical controller for TSA5511 based FM exciter and VCO projects. It combines an Arduino Nano, a 16x2 LCD, three pushbuttons and carefully managed EEPROM settings into a compact front-panel frequency controller.

Designed for TSA5511-based hardware

Originally developed to replace the proprietary controller of the DRFS06 exciter, this project is equally suitable for other TSA5511-based designs.

Menu driven

The LCD interface provides direct access to adjust VCO, PLL, and general settings on the fly.

EEPROM aware

Configurable settings and last-used operating states are restored after restart, while avoiding unnecessary EEPROM writes where practical.

Project overview

This page is a concise landing page for the TSA5511 PLL Controller project. The full source code, revision history and implementation details are maintained in the GitHub repository, while this page summarizes the purpose, hardware concept and main operating features for search engines and visitors.

The controller is intended for VCO frequency control from 64 MHz up to 1,300 MHz, within the specification range of the TSA5511. It supports configurable VCO bands, selectable PLL crystal frequency, I²C address handling, lock/unlock indication, display of station-name and enhanced LCD backlight control.

Main features

  • Arduino Nano or compatible controller platform.
  • 16x2 LCD user interface in 4-bit mode.
  • Three-button operation using DOWN, SET and UP.
  • Double-click access to the configuration menu.
  • Configurable VCO frequency bands.
  • Frequency precision matched to the PLL reference step.
  • Separate memory for operating frequency per VCO band and crystal frequency.
  • Selectable TSA5511 I²C address with verification.
  • Configurable PLL crystal frequency: 1.6 MHz or 3.2 MHz.
  • Charge-pump current configuration for locked state.
  • Output port mapping for lock, unlock and RF-stage control.
  • Station-name editor for the idle display.
  • Optional LCD backlight dimming and manual backlight-off function.
  • I²C communication retry handling and visible fault indication.

Hardware overview

The hardware consists of an Arduino Nano or compatible board, a standard 16x2 LCD, three pushbuttons with 470 nF debounce capacitors, the TSA5511 PLL circuit and an optional PLL lock LED. The lock LED can also act as a blinking fault indicator, while the LCD displays the lock status directly.

Controller Arduino Nano or compatible board
User interface 16x2 LCD, DOWN / SET / UP buttons
PLL device TSA5511 with configurable I²C address
Typical use FM exciter, VCO and RF frequency-control projects

Pull-up resistors on SDA and SCL are required. If the I²C bus passes through RF-decoupling circuitry, lower pull-up values such as 1 kΩ or 2 kΩ may improve communication reliability.

Operation and settings

  • VCO settings: select the active VCO frequency band and choose the displayed/settable frequency precision.
  • PLL settings: configure the I²C address, PLL crystal frequency, charge-pump behaviour and output-port mapping.
  • General settings: edit the station name and enable or disable automatic LCD backlight dimming.
  • Exit menu: explicitly save changes to EEPROM, discard changes or return to the menu.

In normal use, the VCO frequency is adjusted with UP and DOWN and confirmed with SET. Holding UP or DOWN starts an accelerated sweep through the selected frequency band. If a pending frequency change is not confirmed, the controller returns to the main screen unchanged after timeout.
