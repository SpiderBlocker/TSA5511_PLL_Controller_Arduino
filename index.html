TSA5511 PLL Controller for Arduino

A PLL controller for TSA5511 based FM exciter and VCO projects. It combines an Arduino Nano, a 16x2 LCD, three pushbuttons, System Settings, Quick Menu and carefully managed EEPROM storage into a compact front-panel frequency controller.

Originally developed for replacing the proprietary controller of a DRFS06 exciter, while remaining useful for other TSA5511 based designs.

Menu driven

The LCD interface separates persistent System Settings from the direct Quick Menu, so configuration and everyday operation remain clear.

EEPROM aware

Persistent system settings, per-band operating states and user memories are restored after restart, while temporary actions stay out of EEPROM.

Project overview

This project provides a compact Arduino-based front-panel controller for TSA5511 PLL designs. It brings both persistent configuration and everyday operating functions to a 16x2 LCD interface, so frequency control, memory handling and RF-drive control can be handled directly from the hardware.

The controller is intended for VCO frequency control from 64 MHz up to 1,300 MHz, within the specification range of the TSA5511. It supports configurable VCO bands, selectable PLL crystal frequency, I²C address handling, lock indication, user memories, station-name editing, temporary RF-drive control, optional menu-title display and LCD backlight control.

Main features

  • Arduino Nano or compatible controller platform
  • 16x2 LCD user interface in 4-bit mode
  • Three-button operation using DOWN, SET and UP
  • Double-click access to the System Settings menu
  • Long-press access to the Quick Menu
  • Configurable VCO frequency bands
  • Frequency precision matched to the PLL reference step
  • Separate last-frequency memory per VCO band and crystal setting
  • Six user memory slots per VCO band and crystal setting
  • Selectable TSA5511 I²C address with verification
  • Configurable PLL crystal frequency: 1.6 MHz or 3.2 MHz
  • Charge-pump current configuration for locked state
  • Output port mapping for lock, unlock and RF-stage control
  • Station-name editor for the idle display
  • Temporary RF-drive enable/disable control without EEPROM storage
  • Optional LCD backlight dimming and manual backlight-off function
  • Optional menu title display when entering menus
  • Double-confirmed factory reset for stored settings and user memories
  • I²C communication retry handling and visible fault indication

Hardware overview

The hardware consists of an Arduino Nano or compatible board, a standard 16x2 LCD, three pushbuttons with 470 nF debounce capacitors, the TSA5511 PLL circuit and an optional PLL lock LED. The lock LED can also act as a blinking fault indicator, while the LCD displays the lock status directly.

Controller Arduino Nano or compatible board
User interface 16x2 LCD, DOWN / SET / UP buttons
PLL device TSA5511 with configurable I²C address
Typical use FM exciter, VCO and RF frequency-control projects

Pull-up resistors on SDA and SCL are required. If the I²C bus passes through RF-decoupling circuitry, lower pull-up values such as 1 kΩ or 2 kΩ may improve communication reliability.

Operation and settings

  • System Settings: double-click SET to configure VCO, PLL and general settings, then explicitly save or discard pending changes.
  • VCO settings: select the active VCO frequency band and choose the displayed/settable frequency precision.
  • PLL settings: configure the I²C address, PLL crystal frequency, charge-pump behaviour and output-port mapping.
  • General settings: edit the station name, configure LCD backlight dimming, toggle the menu title display or perform a double-confirmed factory reset.
  • Quick Menu: press and hold SET to recall, save or clear user memories, temporarily toggle RF drive, turn the LCD backlight off or return to the main interface.
  • Exit settings: open the save/discard/cancel exit menu to explicitly save changes to EEPROM, discard changes or return to the menu.

In normal use, the VCO frequency is adjusted with UP and DOWN and confirmed with SET. Holding UP or DOWN starts an accelerated sweep through the selected frequency band. If a pending frequency change is not confirmed, the controller returns to the main screen unchanged after timeout. The System Settings menu discards unsaved changes on timeout, while Quick Menu actions are applied immediately.
